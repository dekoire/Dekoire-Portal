Google Ads API Optional – SEO Keyword Validierung
Wozu? Damit werden SEO-Keywords gegen echte Google Suchdaten validiert (Volumen, CPC). Ohne diese Einstellungen läuft die Analyse nur über KI.
Wo bekomme ich das? Alle Schritte: Google Cloud Console → OAuth Client erstellen, dann OAuth Playground für den Refresh Token.
→ ads.google.com/aw/apicenter → Developer Token kopieren (braucht Manager-Konto)
→ Google Cloud → Auth → Clients → OAuth-Client (Typ: Webanwendung) → Client-ID
→ Gleiche Seite wie Client ID → Clientschlüssel (beginnt mit GOCSPX-)
→ OAuth Playground → Zahnrad → eigene Credentials eintragen → Scope: https://www.googleapis.com/auth/adwords → Authorize → Exchange → Refresh Token kopieren
→ Google Ads → oben rechts deine Konto-ID (10-stellig, ohne Bindestriche eintragen)

Speicherort für Bilddateien
Absoluter Pfad oder relativ zum Server-Verzeichnis.
Pro Produkt wird ein Unterordner ID_Titel/ mit dem Originalbild angelegt.
⚠ Python-Bibliothek fehlt. Bitte auf dem Server ausführen:
pip install google-api-python-client google-auth
Die ID aus der Ordner-URL:
drive.google.com/drive/folders/ORDNER-ID-HIER
Den vollständigen JSON-Inhalt der heruntergeladenen Schlüsseldatei hier einfügen.
Einrichtung in 4 Schritten
  1. Google Cloud Console öffnen → Projekt anlegen → Google Drive API aktivieren
  2. IAM & Admin → Dienstkonten → Neues Dienstkonto erstellen → Schlüssel (JSON) herunterladen
  3. Google Drive öffnen → Zielordner → Teilen → Service Account E-Mail eintragen
  4. JSON-Inhalt oben einfügen, Ordner-ID eingeben, speichern & testen

Discord Webhook – Produkte
Was passiert hier? Jedes Mal wenn ein Produkt angelegt oder bearbeitet wird, sendet das System automatisch eine Nachricht an diesen Webhook. Enthalten sind: Titel, Tags, Farben, Bildformat, DPI sowie der Speicherstatus (Excel, Ordner, Supabase).
